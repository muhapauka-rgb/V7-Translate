# Audio Lesson Transcriber RU

Локальное macOS-приложение на Python + PySide6 для расшифровки английских аудиоуроков, перевода на русский и экспорта результата в удобные `Word`- и `PDF`-документы.

Проект делает ставку на простой пользовательский сценарий:

1. Выбрать аудиофайл.
2. Посмотреть его параметры и прогноз времени.
3. Запустить обработку.
4. Получить `.docx`, `.pdf`, а при необходимости ещё `.srt` и `.txt`.

## Архитектура

Приложение разделено на несколько независимых слоёв:

- `app/ui`
  Отвечает за интерфейс: выбор файла, настройку параметров, отображение прогресса и логов.
- `app/core`
  Содержит фонового воркера, пайплайн обработки и механику отмены.
- `app/utils`
  Форматирование времени, размеров файлов и имён выходных файлов.
- `services`
  Бизнес-логика:
  анализ медиафайла,
  оценка времени,
  распознавание,
  перевод,
  экспорт результатов.

## Pipeline

Основной поток обработки устроен так:

1. Выбор файла в интерфейсе.
2. Анализ аудио через `AudioService`.
3. Расчёт прогноза времени через `EstimateService`.
4. Распознавание через `TranscriptionService`.
5. Перевод через `TranslationService` при включённой опции.
6. Экспорт через `ExportService`.
7. Сохранение результата в выбранную папку.

Тяжёлые этапы выполняются в фоне через `QThread`, поэтому интерфейс не блокируется.

## Зависимости

Основные пакеты:

- `PySide6`
- `python-docx`
- `faster-whisper`
- `argostranslate`

Системные зависимости:

- `ffmpeg` / `ffprobe` для точного анализа медиафайлов

## MVP

Что уже входит в первую рабочую версию:

- графический интерфейс на русском языке;
- выбор аудиофайла и папки сохранения;
- отображение имени файла, длительности, размера и прогноза времени;
- выбор качества распознавания;
- фоновая обработка без блокировки UI;
- отмена процесса;
- экспорт в `.docx`, `.pdf`, `.srt`, `.txt`;
- накопление истории реального времени обработки для более точного прогноза;
- архитектура с отдельными сервисами и возможностью заменить движки позже.

Что оставлено на следующий этап:

- пакетная обработка;
- дополнительные языки перевода;
- извлечение ключевых фраз;
- словарь сложных слов;
- AI-сводка урока;
- режим таблицы EN/RU.

## Установка

1. Создайте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Установите `ffmpeg`, если его ещё нет:

```bash
brew install ffmpeg
```

4. Для локального перевода установите языковую модель `Argos Translate` для направления `en -> ru`.

Пакет `argostranslate` ставится через `requirements.txt`, но сама языковая модель ставится отдельно. Пока она не установлена, приложение сможет работать без перевода, если снять чекбокс перевода.

## Запуск

```bash
python run.py
```

## Как проверить MVP

1. Открыть приложение.
2. Выбрать `mp3`, `m4a`, `wav`, `mp4` или `mov`.
3. Убедиться, что появились:
   имя файла,
   длительность,
   размер,
   оценка времени.
4. Выбрать качество и папку сохранения.
5. Нажать `Старт`.
6. Проверить, что по завершении появились файлы экспорта.

## Упаковка в macOS `.app`

В проект добавлен `setup.py` для `py2app`.

Сборка:

```bash
pip install py2app
python setup.py py2app
```

Результат появится в папке `dist/`.

## Примечания по распознаванию и переводу

- Для распознавания используется `faster-whisper`.
- Если нужная модель Whisper ещё не скачана, при первом запуске она будет загружена автоматически.
- Для режима `Максимально точно` приложение сначала пробует более тяжёлую модель, а если это невозможно, откатывается на более лёгкую.
- Если локальный переводчик не настроен, приложение покажет понятное сообщение и предложит либо установить переводчик, либо повторить запуск без перевода.
